# Design Opinions

Personal design philosophy and opinions for code, architecture, and prose. This
file holds long-lived principles and judgment calls that go beyond formatting
and mechanics (those live in `STYLE.md` and `PRACTICES/`). Agents should read
this before writing or reviewing code, and follow these principles unless the
project's own conventions override.

This is a living document. Add, revise, and delete as opinions sharpen. A
short, well-argued rule beats a long hedged one. Each opinion should say *what*
the rule is, *why* it holds, and when (if ever) it does not apply.
